506 /*
507  This structure is used to define bind information, and
508  internally by the client library.
509  Public members with their descriptions are listed below
510  (conventionally `On input' refers to the binds given to
511  mysql_stmt_bind_param, `On output' refers to the binds given
512  to mysql_stmt_bind_result):
513 
514  buffer_type - One of the MYSQL_* types, used to describe
515  the host language type of buffer.
516  On output: if column type is different from
517  buffer_type, column value is automatically converted
518  to buffer_type before it is stored in the buffer.
519  buffer - On input: points to the buffer with input data.
520  On output: points to the buffer capable to store
521  output data.
522  The type of memory pointed by buffer must correspond
523  to buffer_type. See the correspondence table in
524  the comment to mysql_stmt_bind_param.
525 
526  The two above members are mandatory for any kind of bind.
527 
528  buffer_length - the length of the buffer. You don't have to set
529  it for any fixed length buffer: float, double,
530  int, etc. It must be set however for variable-length
531  types, such as BLOBs or STRINGs.
532 
533  length - On input: in case when lengths of input values
534  are different for each execute, you can set this to
535  point at a variable containining value length. This
536  way the value length can be different in each execute.
537  If length is not NULL, buffer_length is not used.
538  Note, length can even point at buffer_length if
539  you keep bind structures around while fetching:
540  this way you can change buffer_length before
541  each execution, everything will work ok.
542  On output: if length is set, mysql_stmt_fetch will
543  write column length into it.
544 
545  is_null - On input: points to a boolean variable that should
546  be set to TRUE for NULL values.
547  This member is useful only if your data may be
548  NULL in some but not all cases.
549  If your data is never NULL, is_null should be set to 0.
550  If your data is always NULL, set buffer_type
551  to MYSQL_TYPE_NULL, and is_null will not be used.
552 
553  is_unsigned - On input: used to signify that values provided for one
554  of numeric types are unsigned.
555  On output describes signedness of the output buffer.
556  If, taking into account is_unsigned flag, column data
557  is out of range of the output buffer, data for this column
558  is regarded truncated. Note that this has no correspondence
559  to the sign of result set column, if you need to find it out
560  use mysql_stmt_result_metadata.
561  error - where to write a truncation error if it is present.
562  possible error value is:
563  0 no truncation
564  1 value is out of range or buffer is too small
565 
566  Please note that MYSQL_BIND also has internals members.
567 */
568
